# Medical Transcription Classifier & Semantic Search Engine

An NLP capstone project that classifies medical transcription notes into
clinical specialties using two fundamentally different approaches:

1. **Supervised Classifier** — TF-IDF vectorization + three ML models
   (Logistic Regression, Naive Bayes, Random Forest)
2. **Semantic Search Engine** — sentence embeddings (MiniLM 384-D) +
   ChromaDB vector store + mini-RAG retrieval-and-vote classifier

Built during an IBM Data Engineering internship using the
[MTSamples](https://www.kaggle.com/datasets/tboyle10/medicaltranscriptions)
medical transcription dataset.

## Key Results

| Approach | Metric | Result |
|----------|--------|--------|
| Logistic Regression | Accuracy / Macro F1 | ~70% / ~66% |
| Naive Bayes         | Accuracy / Macro F1 | ~53% / ~23% |
| Random Forest.      | Accuracy / Macro F1 | ~61% / ~56% |
| Semantic Search | Precision@5 | ~64% (17× random baseline) |

## 💡 Key Insights & Model Evaluation

### 🏆 Why Logistic Regression Won
Logistic Regression was the clear champion, achieving **~70% Accuracy** and a **~66% Macro F1-Score**. Text data vectorized via TF-IDF results in a highly high-dimensional, sparse feature space where classes are often linearly separable. Logistic Regression excelled at drawing clean decision boundaries across these dimensions. Crucially, by utilizing class-weight balancing (`class_weight='balanced'`), the model successfully penalized majority-class errors, allowing it to maintain high predictive power on rare medical specialties.

### 📉 The Limitations of Naive Bayes & Random Forest
* **Naive Bayes (~53% Acc / ~23% Macro F1):** Completely collapsed on the minority classes. Because it relies heavily on the "prior probability" of each class, the extreme class imbalance completely drowned out the feature evidence. It was also crippled by its "naive" assumption of word independence, double-counting highly correlated medical terms (e.g., "blood" and "pressure").
* **Random Forest (~61% Acc / ~56% Macro F1):** While it captured complex, non-linear interactions that linear models missed, it fell short of Logistic Regression. Without extreme tuning, ensemble decision trees naturally tend to split on majority-class features to reduce overall dataset entropy quickly, making them less optimal for this specific high-dimensional text imbalance.

### 🔍 Semantic Search Success
Leveraging dense vector embeddings via `sentence-transformers` and `chromadb`, the Semantic Search pipeline achieved a **Precision@5 of ~64%**. This represents a massive **17× improvement over a random baseline**. This proves that even when rigid classification boundaries are fuzzy, the underlying vector space successfully maps and groups semantically similar medical clinical transcriptions together based on contextual meaning rather than just raw keyword matching.

## How to Run

Run notebooks in order — each builds on the previous:

**Notebook 1 (Phases 1-2):** EDA, data cleaning, statistical analysis
(chi-square, PMI, TF-IDF). Produces `cleaned_medical_notes.csv`.

**Notebook 2 (Phase 3):** Trains three classifiers on TF-IDF features.
Evaluates with Macro F1, confusion matrices, error analysis, and
coefficient interpretability.

**Notebook 3 (Phases 4-5):** Builds the semantic search engine. First
run computes embeddings (~5 min on CPU) and builds the ChromaDB index.
Subsequent runs load from cache instantly. Validates with Precision@5
and retrieval confusion matrix.

> **Note:** Notebook 3's first run downloads the MiniLM model (~80MB,
> one-time) and computes embeddings for ~8,000 text chunks. This takes
> a few minutes on CPU. After that, cached files make it instant.

## Dataset

[Medical Transcriptions (MTSamples)](https://www.kaggle.com/datasets/tboyle10/medicaltranscriptions)
— 4,999 medical transcription notes across 40 specialty labels.

**Key data challenge:** ~52% of transcriptions are cross-listed under
multiple specialties (the same note filed under Surgery AND Cardiovascular
AND Radiology). We addressed this through evidence-based filtering:

- **Non-specialty labels** (SOAP Notes, Letters, etc.) — 96% overlap with
  real specialties → dropped (redundant format labels).
- **Surgery** — 95% cross-listed → dropped (procedural label, not a
  distinct clinical vocabulary).
- **Radiology** — 95% cross-listed → dropped (service specialty; imaging
  reports already exist under the ordering specialty).

Final clean dataset: **~2,857 notes across 26 genuine clinical specialties**.

## Methodology Highlights

### Evidence-Based Cleaning
Every drop decision justified by set-intersection analysis — not
arbitrary thresholds. Chi-square doubled as a data-quality audit,
surfacing de-identification artifacts.

### Three Statistical Methods (Phase 2)
Chi-square (word ↔ specialty dependence), PMI (meaningful phrases vs
random pairings), TF-IDF (word uniqueness scoring) — all three converge
on separating signal from noise.

### Parametric vs Non-Parametric
Phase 3 trains models that learn parameters and discard the data.
Phase 4 stores the data and references it at query time. Comparing
both on the same dataset reveals their complementary strengths.

### Honest Evaluation
Macro F1 over accuracy (imbalance-aware). Held-out validation set for
the search engine (prevents self-retrieval). Error analysis on
confident-but-wrong predictions.
